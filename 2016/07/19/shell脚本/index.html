shell脚本的编写

博客
  • linux
  • shell
  • 服务器

背景

遇到一个业务需求，几台服务器需要从某个主服务器拉取文件。同时需要备份之前的文件。前面N个月，一直是手动进行的。后来实在受不了，想想还是需要使用程序来解决。
百度一下，什么自动化部署等高度智能化，有点高大上，用不来。于是自行百度，写点脚本去解决问题。

问题描叙

脚本的功能大致如下：

  1. 备份服务器当前的文件。按照日期备份 2016-07-19project_name
  2. 拉取指定的服务器的文件到本地。
  3. 避免重复执行。

学习资源

我也是第一次写脚本。懂一点。平时都是简单的 ls rm cp等。这一次是要写入sh文件运行。算是一个小小的升级吧。

  1. 入门级别的资源 http://www.runoob.com/linux/linux-shell.html shell教程|菜鸟教程

该网址基本满足了我的本次需要。

解决思路

整理下思路，大致如此

  1. 先定义几个变量。变量赋值

  2. 备份 使用cp命令

  3. 拉取远程的文件。使用scp命令

一些心得与注意事项

  1. 变量赋值，=号左右两侧禁止空格。如果有空格会报错。写习惯了php，老喜欢添加空格。导致一开始就出错。
    正确的例子
    home_dir="/home/sites"
    错误的例子
    home_dir = "/home/sites"

  2. 表达式需要添加符号 这里我使用到一个日期的表达式。需要复制给变量。例如下面的 date +%Y-%m-%d 两侧 需要添加该符号bak_name=date +%Y-%m-%d${project_name}`

  3. 数组的获取。使用@ 或 * 可以获取数组中的所有元素。直接使用数组名，只能获取第一个元素。

  1. 复习了下其他的命令。记录一下，加深记忆。

    ：set nu 显示vi行号

    ：set nonu 隐藏vi行号

    if [ ! -f "$myPath"]; then -f表示 判断文件是否存在 -d表示判断目录是否存在 -x表示是否存在并具有权限

  2. 部署到服务器运行时，还需要运行chmod +x ./1.sh 确保有执行权限

  3. 如果sh文件在window上编辑过，上传到服务器上还需要转格式。否则报错。使用vi 1.sh 打开以后执行下面的代码

    ：set ff=unix 即可
