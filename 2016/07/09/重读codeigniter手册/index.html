阅读CodeIgniter手册(一)

博客
  • CodeIgniter
  • php

写着代码，突然感觉迷惘了。到底我会什么。之前在合肥用的thinkphp，去中新用的公司自己的php框架。来杭州又开始CodeIgniter 最近又看了点yii框架。
写着天天平凡的代码，感觉活的太苦闷了。
经历太分散，我需要集中。需要去解决CodeIgniter开源的一些问题。
一些从手册开始。好好看一遍手册，然后去阅读源码。精通一门框架，为工作，为水平而努力
今天主要阅读了

  • CodeIgniter URL
  • 控制器
  • 错误处理
  • 控制器
  • 保留名称
  • URL路由
  • 错误处理
    -以CLI方式运行
    这些主题。认识到一些心得，之前不曾注意过的
  1. site_url 建议产生页面url base_url 建议用来访问文件或者图片
  2. 控制url 是否为普通的url 。尽量避免，使用框架的url分段模式。否则很多地方无法使用 
    `$config['enable_query_strings']=FALSE;`
  3. 文件名与类名的问题。 这里指的是控制器。文件名与类名需要大写。 控制器名称与模型名称有区别。模型需要带model
  4. CodeIgniter 的路由很强大。可以自定义，支持正则和函数调用
    `// $route['product/([a-zA-Z]+)/edit/(\d+)'] = function ($product_type, $id){
    return ‘welcome/add/‘ . strtolower($product_type) . ‘/‘ . $id;
    };`
  5. CodeIgniter 路由可以使用http动词。在处理restful时特别好用。
    HTTP 动词（GET、PUT、POST、DELETE、PATCH）
    $route['products']['put'] = 'product/insert';
    $route['products']['put'] = 'product/insert';
    如果向products控制器发送 发送Put请求，则会执行product控制器的insert方法。
    第二个例子表示 像products+数字组合的url发送时，才执行。
  6. 日志功能需要重视起来。日志的开关与文件夹设置在config文件中
    `log_message($level, $message, $php_error = FALSE)
    //很重要。调试日志。要求文件夹可写。并开启开关。否则不会产生文件产生的文件是默认是 2016-07-09 16:16:14.php 日志的配置位于。1表示只记录错误信息 0表示禁止日志 默认的日志目录位于根目录下Log文件夹$config['log_threshold'] = 1;$config['log_path'] = '';
  7. 基准测试。主要是时间、内测测试。优化程序使用
    `$this->benchmark->mark('code_start');
    $this->benchmark->mark(‘code_end’);
    echo $this->benchmark->elapsed_time(‘code_start’, ‘code_end’);`
  8. 程序分析类库的使用 。方便追踪问题
    `$this->output->enable_profiler(TRUE);`
  9. 网页缓存。
    `$this->output->cache($n);`
    表示缓存页面$n秒 过期自动删除或者需要手动删除。 缓存文件夹要具 备可写权限。只能缓存从view输出的内容。
  10. 控制器的私有方法。添加private即可无法被访问。或者_xxx 也可以。

  11. 重新映射 remap方法 表示可以重新映射。例如上面的例子表示访问view方法会被跳转到del_all执行。
    `public function _remap($method, $params = array())
    {

    $method = 'process_'.$method;
 if (method_exists($this, $method))
 {
     return call_user_func_array(array($this, $method), $params);
 }
 show_404();

    }
    `
    这个例子表示。所有访问的方法，会自动添加process_前缀。 然后判断是否存在。存在则执行该方法。

  12. $route['translate_uri_dashes'] = FALSE; 该配置没啥用。
    其主要的效果，就是如果 my-products 则会自动转换为my_product
    中横线转为下划线。因为连字符无法出现在类名与方法名中。 可以通过该配置实现url中有连字符。

  13. index.php入口文件。控制 app目录，系统目录，还有视图目录的安全。

    可以实现代码的安全
